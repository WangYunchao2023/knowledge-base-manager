## guidance-web-access Skill 演进 (v3.0 → v3.5)

### 核心架构变更

**旧架构**：Cortana接到任务 → 脚本自动分析(extract_task_intent/match_override) → 执行
**新架构**：Cortana接到任务 → Cortana分析任务、制订策略 → 传入cortana_execute_flow()执行

### 两种执行模式

| 模式 | 入口函数 | 适用场景 |
|------|----------|----------|
| 有经验 | `cortana_execute_flow()` | URL/导航路径已知，Cortana直接传入 |
| 无经验 | `cortana_perception_flow()` | URL未知，Cortana自感知页面结构 |

### 关键函数

```
web_access.py (v3.5.0)
├── log()
├── generate_truncated_variants()
├── get_links_by_text_content_v2()    # 内容扫描
├── smart_interact()                  # 关键词输入
├── explore_with_pagination_v2()     # 探索+翻页
├── fuzzy_semantic_filter()          # 已废弃/死代码
├── final_download()
└── cortana_execute_flow()          # Cortana主导入口
└── cortana_perception_flow()       # 自感知页面结构入口
```

### 删除的函数（v3.4.0清理）

- `main_flow()` - 旧入口
- `explore_with_pagination()` - 旧探索函数
- `get_user_overrides()` - 经验匹配
- `match_override()` - 经验匹配
- `extract_task_intent()` - 语义解析
- `extract_var_from_match()` - 变量提取
- `request_translation()` - 翻译请求

### filter_criteria vs fuzzy_semantic_filter

| 名称 | 类型 | 说明 |
|------|------|------|
| `filter_criteria` | **参数**（列表） | Cortana制订的过滤条件，Python代码执行，不消耗token |
| `fuzzy_semantic_filter()` | **函数** | 旧版过滤逻辑，已废弃 |

### 使用示例

```bash
# 有经验模式（URL已知）
python web_access.py --cortana-plan '{
  "task":"下载沟通交流相关指导原则",
  "search_url":"https://www.cde.org.cn/zdyz/fullsearchpage",
  "search_var":"沟通交流",
  "filter_criteria":["沟通交流"],
  "method":"search_only",
  "save_dir":"~/Documents/工作/法规指导原则/沟通交流"
}'

# 无经验模式（自感知）
python web_access.py --perception '{
  "task":"下载某类指导原则",
  "base_url":"https://www.cde.org.cn",
  "search_var":"某关键词"
}'
```

### 经验可包含的内容

- URL（搜索页、列表页）
- 导航路径（如：先进入"指导原则"→再进入"发布通告"）
- 操作步骤（点击哪个元素、填什么）
- 选择器
- 执行方式（search_only/navigate_only/both）

## Skill 创建 - session-save

已创建会话保存 Skill：`~/.openclaw/workspace/skills/session-save/`

**功能：**
- 自动判断会话类型（Skill优化类 / 常规）
- Skill优化类 → 保存到 `skills/session-save/优化过程对话记录/`
- 常规类 → 保存到 `~/Desktop/`
- 文件名格式：`类型-开始时间-结束时间.html/md`

**使用方式：**
```bash
python3 ~/.openclaw/workspace/skills/session-save/scripts/save_session.py <session_key>
```

**关键改进（教训）：**
- OpenClaw 的 sessions_history API 会丢失格式（表格、ASCII图）
- 必须直接从 JSONL 文件读取原始内容
- JSONL 文件位置：`~/.openclaw/agents/main/sessions/`
- 通过 sessions.json 查找 session key 到 JSONL 的映射
