## 8. opendataloader-pdf skill 重构（v1.8 → v2.0）

### 核心变更：逐页自适应处理

**旧架构**：整文档一刀切（全 OCR 或全文字提取），采样误判率高
**新架构**：逐页独立判断，每页单独选择工具处理

```
输入 PDF → classify_pages() 逐页检测
    ↓
digital页 → opendataloader --pages N,N...
scanned页 → qwen_ocr_pdf(pages_to_skip=digital_pages)
    ↓
merge_per_page_results() 按 page_number 合并 → 统一 kids 结构
```

### 关键修复

| 修复项 | 说明 |
|--------|------|
| **逐页分类** | `classify_pages()`：每页用 pypdf 提取文字量（>50字=digital），返回 `digital_pages[]` + `scanned_pages[]` |
| **VRAM 协调** | `qwen_ocr_pdf()` 内置 try/finally：`acquire_for_comfy()` → OCR → `release_and_restore()` |
| **DOCX kids 统一** | `extract_word_to_json()` 输出 `kids[]` 结构（兼容 PDF），与 `compute_page_hash()` 兼容 |
| **DOCX fallback** | 合并失败（0元素）时直接用纯 docx 内容，避免空输出 |
| **JSON 读全文** | `load_index()` 读完整 JSON，避免 500 字符截断导致解析失败 |
| **PDF 字节哈希保底** | 全扫描文档用 `pdf_byte` 哈希替代文本哈希，避免 0 内容哈希 |
| **页码分配** | Stage1（前50字匹配页码文本）→ Stage2（顺序分配到未匹配元素） |

### VRAM 协调机制

**路径**：`~/.openclaw/scripts/vram_manager.py`（系统级工具）

**调用方式**（opendataloader-pdf 已内置）：
```python
sys.path.insert(0, "/home/wangyc/.openclaw/scripts")
from vram_manager import VMgr
vm = VMgr()
vm.acquire_for_comfy(reason="opendataloader-qwen-ocr")  # 腾出VRAM

```

**CLI 方式**（跨 session 调用）：
```bash

python3 ~/.openclaw/scripts/vram_manager.py release    # 恢复
python3 ~/.openclaw/scripts/vram_manager.py status      # 状态
```

### 教训

- **逐页处理优于整文档一刀切**：混合 PDF 在文档中任何位置穿插扫描页都可能，逐页判断是唯一正确方案

- **DOCX JSON 必须输出 kids 结构**：knowledge_base_manager 的 `compute_page_hash` 依赖 `kids` 字段
