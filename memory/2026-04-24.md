## 7. impurity-data-check skill 重大重构（v0.9.3 → v0.9.4）

### 核心变更：Agent 判断模式

**之前**：固定代码自动匹配项目配置（`auto_match_project` 关键字匹配）  
**现在**：提取项目信息 + 显示可用配置 → **由 Agent 判断**使用哪个项目配置

**关键代码修改**（checker.py check_file 函数）：
- 去掉 `resolver_for_proj.auto_match_project(project_info)` 自动匹配
- 无 `--agent-decision` 时：打印提取结果 + 可用配置列表，提示 Agent 判断
- 有 `--agent-decision` 时：直接使用 Agent 指定的配置

### v0.9.4 修复：ProjectInfoExtractor 从 Excel 内容提取

**问题**：`r'([\\u4e00-\\u9fa5]'` 在 raw string 中 `\u` 被解释为字面反斜杠+u，而非 unicode 转义

**修复**：改为非 raw string `'([\u4e00-\u9fa5]'`

**提取结果对比**：
```
之前（只从文件名）：项目编号=LMS002, 药品名=未提取到, 产品名=未提取到
现在（从文件名+Excel内容）：
  项目编号: LMS002
  检测项目: 有关物质
  药品名: 黄体酮          ← 从Excel A1单元格提取
  产品名: 黄体酮预灌封   ← 从Excel A1单元格提取
```
