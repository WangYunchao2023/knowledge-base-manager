# 2026-04-04 日志

## stability-planner v5 版本管理逻辑最终确认

### 任务标识定义
- **同一任务的判断**：药品名 + 批号 + 条件（三者都相同）
- **新会话 = 新任务**（默认），每次生成新的稳定性研究计划
- **跨会话不跟踪**：不再依赖 `.session_last_plan.txt` 做任务识别

### 生成行为
1. **新会话下达任务**：扫描磁盘查重（药品+批号+条件）
   - 无重复 → 直接生成
   - 有重复 → ⚠️ 提醒用户，由用户决定
2. **同会话内优化**：Agent 在同一对话流中直接生成同名文件（覆盖）

### 文件命名
```
{药品名}_{批号}_{条件}_稳定性计划.xlsx
```
- 批号中逗号转下划线，空格去除
- 条件中 `+` 转 `_`，`℃` 转 `C`，空格去除

### 批号区分策略（最终设计）
- 同一药品 + 不同批号 = 不同研究任务，各自独立文件
- 同一药品 + 同一批号 + 不同条件 = 不同研究任务（因为条件在文件名中不同）
- 同一药品 + 同一批号 + 同一条件 = 同一任务（优化时覆盖）
