    方案 C：内容哈希（推荐）                                                           
                                                                                   
在入库时先用 opendataloader                                                        
提取纯文本内容，再对文本内容算哈希——排版差异和标注不影响文本内 容哈希值：          
                                                                                   
```                                                                                
  文件到达 → opendataloader 提取纯文本 → 内容哈希 → 存 guidance_index.json         
                                                      ↓                            
                                内容哈希相同 → 提示"与已有文档内容相同，跳过"      
                                内容哈希不同 → 正常入库                            
```                                                                                
                                                                                   
优点：排版、标注、页眉页脚等差异不影响去重                                         
前提：opendataloader 能正确提取纯文本（目前已满足）




1、内容哈希值对比，能确保排版不同时（实际使用中情况可能很复杂，特别是有表格时，很可以因提取出来的表格不一致），提取的内容的一致性吗？是否可以修改为提取正文的第一页，第二页的内容，仅对这两页的内容进行内容哈希值的对比。更好的方式，先提取第一页的进行内容哈希值对比，然后再提取第二页的进行内容哈希值对比，这样下来，两页都相同时，就可以认为是相同 的。
