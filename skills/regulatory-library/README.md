# 💊 Regulatory Library Skill - 开发上下文与架构规范

## 🎯 核心定位 (Design Philosophy)
本 Skill 作为一个 **Sidecar (辅助参考工具)** 存在，为实验方案生成、申报资料撰写、汇报撰写等模块提供法规依据。
- **定位**：参考/指导作用，而非替代撰写。
- **原则**：全本地保存、.md/.json 结构化、开源免费工具链、高可扩展性。

## 🏛️ 物理保存结构 (Directory Structure)
必须严格遵循以下三层嵌套逻辑：

### **L1: 机构层 (Agency)**
- `NMPA_CDE/` (中国国家药监局与药审中心合并存放)
- `ICH/` (国际技术协调会)
- `FDA/` (美国 FDA)
- `EMA/` (欧洲 EMA)
- ... (可横向扩展)

### **L2: 专业领域层 (Domain)**
在机构目录下，按研发专业领域划分：
- `00_General/` (跨专业的通用法规，如《药品管理法》)
- `CMC/` (药学研究：制剂、分析、生产)
- `Pre-Clinical/` (非临床：动物安评、毒理)
- `Clinical/` (临床研究：BE、I/II/III期)
- `Regulatory/` (注册申报要求：文件撰写规范)

### **L3: 药物类型层 (Category/Drug Type)**
在专业领域目录下，进一步区分通用与专用：
- `00_General/` (领域内通用，如 CMC 下的《稳定性研究指导原则》)
- `Chemical/` (化学药专用)
- `Biological/` (生物药专用)
- `TCM/` (中药专用)

## 🔄 核心运行逻辑
- **通用性处理**：指导原则若适用于多种药物，优先放入该层级的 `00_General`。
- **兜底机制**：
  1. 优先检索 `02_Reference/*.md` 摘要（带页码追溯 `[Source: Pxx]`）。
  2. 若摘要有遗漏，自动调用 `ripgrep` 对 `01_Sources/` 的原始文本底稿进行全文扫描。
- **跨模块调用**：通过检索 `.json` 元数据标签，为“药品数据库”提供法规参数比对。
