# impurity-data-check

HPLC有关物质检测数据明显错误检查工具。

## 报告限配置

**阈值优先级（从高到低）：**

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1 | `--threshold` 命令行参数 | 全局单一阈值 |
| 2 | `project_configs` 项目级配置 | 按文件名/路径关键词匹配 |
| 3 | `default_threshold` / `impurity_thresholds` | 全局默认 |
| 4 | Excel 表头自动识别 | 扫描"报告限：X%"等关键词 |
| 5 | 内置默认值 | 0.05% |

## 配置文件

放置于 **skill 目录**：`~/.openclaw/workspace/skills/impurity-data-check/impurity-checker.yaml`

```yaml
# ── 项目级配置（按文件名或路径关键词匹配） ──
project_configs:

  # LMS002 项目
  LMS002:
    default_threshold: 0.05
    impurity_thresholds:
      "杂质M": 0.05
      "杂质N": 0.10
      "其他单个杂质": 0.05

  # 可添加更多项目
  # LMS001:
  #   default_threshold: 0.10
  #   impurity_thresholds:
  #     "杂质A": 0.10

# ── 全局默认配置（无项目匹配时使用） ──
default_threshold: 0.05
impurity_thresholds:
  "杂质M": 0.05
```

**匹配规则：**
- 文件名或路径包含项目关键词（如 `LMS002`）即匹配
- 匹配到项目 → 使用该项目专属配置
- 未匹配任何项目 → 使用全局默认配置

## 批号格式规范

标准格式：**项目编号-两位年两位月两位日-两位序号**

```
LMS002-260401-05
```

示例：
- `LMS002-260401-05` ✅ 合规
- `LMS002-26040105` ❌ 缺少连字符

## 检查项目

### 1. 批号格式一致性检查

- 检测同一文件内批号格式是否统一
- 识别缺少连字符的批号

### 2. 忽略不计判断合理性检查

- 标记"忽略不计"的含量值必须小于该杂质的报告限
- 含量 ≥ 该杂质的报告限却标"忽略不计" → 报错

## 使用方法

```bash
python3 scripts/checker.py <文件路径> [--threshold <阈值>]
```
