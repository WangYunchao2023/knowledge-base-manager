# check-inspection-data

药物跟检数据错误检查工具。自动扫描 Excel 文件，检测批号格式不规范、日期填写错误等常见填写问题，并输出带修改建议的错误报告。

## 使用方法

### 基本用法

```markdown
/check-inspection-data <文件路径>
```

### 示例

```
/check-inspection-data ~/Documents/工作/稳定性跟检/2024年度稳定性跟检数据表.xlsx
```

### 输出

- ✅ **检查完成**：未发现错误
- ⚠️ **发现 X 个潜在问题**（详见报告）
  - [批号格式] ABC123 → 建议格式：YYYYMMDD-XXX（例：20240401-001）
  - [日期错误] 2024-02-30 → 日期不存在，建议修正为 2024-03-01 或 2024-02-29

---

## 检查规则

### 1. 批号格式检查

| 规则 | 说明 | Severity |
|------|------|----------|
| 批号长度异常 | 批号长度不在合理范围（6-20字符） | WARNING |
| 批号包含非法字符 | 含有空格、特殊符号（除-、/外） | WARNING |
| 批号格式不符合规范 | 常见格式：日期+序号、字母+数字组合等 | INFO |
| 批号疑似写反 | 如"241-01"疑似应为"2024-01" | ERROR |

### 2. 日期检查

| 规则 | 说明 | Severity |
|------|------|----------|
| 日期不存在 | 如 2024-02-30（2月没有30日） | ERROR |
| 日期格式不统一 | 混用 2024-01-01 / 2024.01.01 / 20240101 | WARNING |
| 日期超出合理范围 | 早于2000年或晚于10年后 | WARNING |
| 逻辑矛盾 | 结束日期早于开始日期 | ERROR |
| 取样日期早于生产日期 | 跟检数据常识性错误 | ERROR |

### 3. 通用数据检查

| 规则 | 说明 | Severity |
|------|------|----------|
| 必填字段为空 | 关键字段（批号、日期、检测项）空白 | ERROR |
| 数值超范围 | 温度、pH等超出合理范围 | ERROR |
| 填写不完整 | 存在明显空白区域 | WARNING |

## 配置

可通过 `~/.openclaw/workspace/skills/check-inspection-data/config.yaml` 自定义：

```yaml
batch_rules:
  min_length: 6
  max_length: 20
  allowed_chars: "ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789-/"

date_rules:
  min_year: 2000
  max_year_offset: 10
  allowed_formats:
    - "%Y-%m-%d"
    - "%Y/%m/%d"
    - "%Y.%m.%d"

severity_levels:
  ERROR: 高优先级，必须修正
  WARNING: 中优先级，建议修正
  INFO: 低优先级，仅供参考
```

### 详细报告（Excel）

生成 `检查报告_原文件名.xlsx`，包含：
- Sheet1：错误汇总（位置、当前值、问题描述、修改建议、严重级别）
- Sheet2：错误分布热力图（直观显示问题集中区域）

---

## 注意事项

- 支持 `.xlsx` / `.xls` 格式
- 支持多 Sheet 文件，自动扫描所有 Sheet
- 批号格式规则支持企业自定义（通过 config.yaml）
- 混杂在文本中的日期（如"生产日期：2024-01-15"）也能识别
